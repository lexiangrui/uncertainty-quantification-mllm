  方案 1：Gradient-based Token Importance（梯度 × 输入）（推荐）

  思路：对逐 token 熵做一次反向传播，用梯度幅值衡量每个视觉 token 对预测不确定性的因果贡献。

  importance(v_i) = || ∇_{e_i} H(y|v,t) · e_i ||

  其中 e_i 是第 i 个视觉 token 的嵌入向量，H(y|v,t) 是原图条件下的预测熵。

  流程：
  1. 第一次前向，计算响应 token 的逐 token 熵
  2. 对熵做 backward，提取视觉 token 嵌入上的梯度
  3. 用 ||gradient × embedding|| 的 L2 范数排序，取 top-k
  4. 将 top-k token 的 attention mask 置零，做第二次前向

  # 伪代码
  outputs = model(input_ids=full_ids, pixel_values=pixel_values, return_dict=True)
  logits = outputs.logits[:, prompt_len-1:-1]
  entropy = -(probs * torch.log2(probs + 1e-9)).sum(-1).mean()

  # 反向传梯度到 input embeddings
  entropy.backward()
  # 在 vision token 区间取梯度范数
  grad_norm = input_embeddings.grad.norm(dim=-1)
  vision_grads = grad_norm[first_pos:last_pos]
  _, topk_indices = torch.topk(vision_grads, k)

  优点：
  - 与 Flash Attention 2 完全兼容（梯度通过 fused kernel 正确传播）a
  - 比注意力权重更接近因果意义上"token 到底有没有贡献"
  - 计算成本可控（一次 backward pass）

  缺点：
  - 比纯注意力的 eager 模式多了一次 backward（eager 只需一次带 output_attentions 的前向）
  - 梯度信号可能比注意力噪声大（需要做平滑或 averaging）
  - 需要包装 torch.no_grad() 外的计算逻辑

  与 VAUQ 原始意图的对应关系：
  - VAUQ 原始：注意力高 → 模型"在看"这些区域 → mask 掉
  - Gradient 版本：梯度大 → 这些区域的嵌入对最终熵贡献大 → mask 掉
  - 梯度版测量的是因果贡献，注意力测的是统计关联——梯度在方法论上可能更硬
