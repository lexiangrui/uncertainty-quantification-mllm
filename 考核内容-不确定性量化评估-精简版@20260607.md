## 二、基本背景

多模态大模型可能会生成与图像、题目、上下文或事实不一致的内容，这类现象通常称为多模态幻觉。

需要注意的是，幻觉不等于所有错误。

例如：

* 如果模型只是算错、推理链出错或公式用错，这通常是一般错误或推理错误；
* 如果模型编造了图像中不存在的物体、属性、关系、数字、条件或证据，这才更接近幻觉；
* 如果模型在推理过程中引入题目没有给出的前提、规则或事实，也可以视为 reasoning hallucination。

### 2. 最低实验要求

以下要求用于保证实验具有基本的可比性和可信度：至少能够观察方法在不同数据集、不同模型和不同评价维度上的表现，而不是只在单一设置下得到偶然结论。

请满足以下最低要求：

| 项目 | 最低要求 | 用途 / 考察点 |
| --- | --- | --- |
| 数据集 | 至少 2 个，其中至少包含 1 个幻觉相关 benchmark 和 1 个多模态推理、问答或图表理解类 benchmark | 覆盖不同任务场景，避免只在单一任务上得出结论 |
| 模型 | 至少 3 个模型，其中至少 1 个闭源模型、2 个开源模型 | 比较不同能力水平和开放程度模型的不确定性与幻觉风险差异 |
| 不确定性量化方法 | 至少 3 种，需覆盖至少 2 类不同思路 | 比较不同 UQ 思路的有效性，而不是依赖单一 uncertainty score |
| 幻觉风险评估指标 | 至少 3 个，需覆盖排序、分类或相关性分析中的至少 2 类 | 评估 uncertainty / risk score 是否能用于区分幻觉或高风险样本 |
| 校准方法 | 至少 1 种 post-hoc calibration 方法 | 检验模型置信度是否可以通过后处理变得更可信 |
| 校准指标 | 至少 2 个 | 衡量置信度与真实正确率或可信度之间的匹配程度 |
| 人工标注 | 50-100 条样本 | 提供人类判断参照，用于分析模型置信度与人类可信度感知是否一致 |
| 人类一致性指标 | 至少 2 个相关性或一致性指标 | 衡量模型 confidence / uncertainty 与人类 trust score 的一致程度 |
| 基础结果 | Accuracy、Hallucination Rate、不同模型之间的对比 | 建立基本性能背景，避免脱离模型能力讨论 uncertainty 和 hallucination |
| 案例分析 | 至少 3 个具体案例 | 检验学生是否能解释方法失效、过度自信和高风险样本等现象 |

如果算力、时间或 API 成本有限，可以使用数据集子集，但需要说明采样方式和限制，以及为什么这个子集可以衡量给定的指标。

可选数据集可以参考 LURE、MathVista、MMHal-Bench、POPE、ChartQA、MMMU、MMStar 等。

可选指标可以参考 AUROC、AUPRC、F1 Score、Accuracy、Hallucination Rate、ECE、Brier Score、Pearson Correlation、Spearman Correlation、high-confidence error rate 等。

你需要在报告中说明为什么选择这些数据集、模型、方法和指标。
